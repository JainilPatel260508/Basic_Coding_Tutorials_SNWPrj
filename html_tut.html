Welcome to HTML Basics

Welcome to the HTML Basics Course! This course is designed for complete beginners who want to learn how to create web pages from scratch. HTML, which stands for HyperText Markup Language, is the foundation of every website you see on the internet.

Think of HTML as the skeleton of a website. Just like our skeleton gives structure to our body, HTML gives structure to web pages. It tells the browser what content to display and how to organize it. Whether it's text, images, links, or videos, HTML is what makes it all appear on your screen.

In this course, you'll start with the very basics and gradually build your skills. We use simple language and provide plenty of examples so you can see exactly how everything works. By the end, you'll be able to create your own web pages with confidence!

1. Understanding HTML Structure

Every HTML document follows a basic structure. Think of it like building a house - you need a solid foundation and framework before adding walls and decorations. Let's look at the essential parts of an HTML document.

The Basic HTML Template

Here's what every HTML page needs to have: 

<!DOCTYPE html>
<html>
<head>
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>This is my first paragraph.</p>
</body>
</html>

Breaking It Down

Let's understand what each part does:

  • <!DOCTYPE html> - This tells the browser that this is an HTML5 document. It's like saying "Hey browser, I'm about to give you HTML code!"
  • <html> - This is the root element that wraps all the content on your page. Everything goes inside this.
  • <head> - This section contains information about your page that doesn't appear directly on the screen, like the page title.
  • <title> - This sets the title that appears in your browser tab. It's what people see when they have your page open.
  • <body> - This is where all your visible content goes. Everything you want people to see should be inside the body tags.
Important Note: Notice how tags come in pairs? You have an opening tag like <body> and a closing tag like </body>. The closing tag has a forward slash. Everything between these tags is affected by them.

2. Working with Headings and Paragraphs

Now that you understand the basic structure, let's add some content! Headings and paragraphs are the most common elements you'll use on web pages.

HTML Headings

HTML has six levels of headings, from <h1> to <h6>. Think of them like chapter titles and subtitles in a book. The <h1> is the most important (like a book title), and <h6> is the least important (like a small subtitle). 

<h1>This is a Main Heading</h1>
<h2>This is a Subheading</h2>
<h3>This is a Smaller Subheading</h3>
<h4>Getting Even Smaller</h4>
<h5>Almost the Smallest</h5>
<h6>The Smallest Heading</h6>

How It Looks:

This is a Main Heading

This is a Subheading

This is a Smaller Subheading

Paragraphs

Paragraphs are created using the <p> tag. They're perfect for regular text content, just like the paragraphs in this lesson. Each paragraph automatically gets some space above and below it to separate it from other content. 

<p>This is my first paragraph. It can contain multiple sentences about any topic.</p>
<p>This is my second paragraph. Each paragraph automatically gets some space above and below it.</p>
<p>This is my third paragraph. You can have as many paragraphs as you want.</p>
Pro Tip: Use only one <h1> per page, usually for your main title. This helps search engines understand what your page is about and improves your search rankings!

3. Text Formatting and Styling

Sometimes you want to emphasize certain words or make text stand out. HTML provides several tags for formatting text within paragraphs. These tags help you highlight important information and make your content more visually interesting.

Common Text Formatting Tags 

<p>You can make text <strong>bold and important</strong> using the strong tag.</p>

<p>You can make text <em>italic and emphasized</em> using the em tag.</p>

<p>You can <u>underline text</u> using the u tag.</p>

<p>You can create <small>smaller text</small> using the small tag.</p>

How It Looks:

You can make text bold and important using the strong tag.

You can make text italic and emphasized using the em tag.

You can underline text using the u tag.

Line Breaks

Sometimes you need to break text onto a new line without starting a whole new paragraph. For this, use the <br> tag. This is useful for addresses, poetry, or any content where line breaks matter. 

<p>First line of text<br>
Second line of text<br>
Third line of text</p>

4. Creating Links

Links are what make the web "web-like" - they connect pages together! The <a> tag (which stands for "anchor") is used to create links. Without links, the internet would just be isolated pages. Links are the glue that holds everything together.

Basic Link Structure

A link has two main parts: the destination (where it goes) and the text people click on. The destination is specified in the href attribute. 

<a href="https://www.google.com">Click here to visit Google</a>

<a href="about.html">Go to About Page</a>

<a href="contact.html">Contact Us</a>

The href attribute tells the browser where to go when someone clicks the link. You can link to:

  • External websites - Use the full web address like "https://www.google.com"
  • Other pages on your site - Just use the filename like "about.html"
  • Email addresses - Use "mailto:" like <a href="mailto:email@example.com">Send Email</a>
Remember: Links appear blue and underlined by default. When you hover over them, the cursor changes to a pointing hand, indicating they're clickable.

5. Adding Images

Images make websites more interesting and visually appealing. The <img> tag is used to add images to your web pages. A picture is worth a thousand words, and images can really enhance your website's appearance and user experience.

Image Syntax

Unlike most HTML tags, the image tag doesn't have a closing tag. It's self-closing! The browser knows when it encounters an image tag that it's a complete element. 

<img src="photo.jpg" alt="A beautiful sunset">

<img src="logo.png" alt="Company Logo" width="200">

<img src="https://example.com/image.jpg" alt="Online image">

Important Image Attributes

  • src - The source (location) of your image file. This tells the browser where to find the image. This is required!
  • alt - Alternative text that describes the image. This shows if the image can't load, and helps people using screen readers. Always include this!
  • width and height - Control the size of your image in pixels. You can use just one, and the browser will adjust the other to maintain the original proportions.
Good Practice: Always include alt text for your images. It makes your website more accessible to people with visual impairments and helps with search engine optimization. Write descriptive alt text that explains what the image shows.

6. Creating Lists

Lists are perfect for organizing related items, steps in a process, or any group of information. HTML supports two main types of lists. Using lists makes your content more organized and easier to read.

Unordered Lists (Bullet Points)

Use unordered lists when the order doesn't matter, like a shopping list or a collection of features. 

<ul>
    <li>Apples</li>
    <li>Bananas</li>
    <li>Oranges</li>
    <li>Grapes</li>
</ul>

How It Looks:

  • Apples
  • Bananas
  • Oranges
  • Grapes

Ordered Lists (Numbered)

Use ordered lists when sequence matters, like instructions or rankings. The browser automatically numbers them for you. 

<ol>
    <li>Open your HTML editor</li>
    <li>Write your HTML code</li>
    <li>Save the file as index.html</li>
    <li>Open it in a web browser</li>
</ol>

How It Looks:

  1. Open your HTML editor
  2. Write your HTML code
  3. Save the file as index.html
  4. Open it in a web browser

7. Building Simple Tables

Tables are useful for displaying data in rows and columns, like schedules, pricing lists, or comparison charts. Tables help organize information in a clear, easy-to-read format.

Basic Table Structure

A table consists of rows (<tr>) and cells. Cells can be headers (<th>) or regular data (<td>). Header cells appear bold to distinguish them from regular data. 

<table border="1">
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>City</th>
    </tr>
    <tr>
        <td>John</td>
        <td>25</td>
        <td>New York</td>
    </tr>
    <tr>
        <td>Sarah</td>
        <td>30</td>
        <td>London</td>
    </tr>
</table>

In this example:

  • <table> creates the table
  • <tr> creates each row (row = horizontal line)
  • <th> creates header cells (they appear bold and centered)
  • <td> creates regular data cells
  • border="1" adds borders so you can see the table structure

8. Creating Forms for User Input

Forms allow visitors to send information to you. They're used for contact forms, login pages, surveys, newsletter signups, and much more. Forms are essential for collecting user data.

Basic Form Structure 

<form>
    <label>Your Name:</label>
    <input type="text" name="username">

    <label>Your Email:</label>
    <input type="email" name="email">

    <label>Message:</label>
    <textarea name="message"></textarea>

    <button type="submit">Send Message</button>
</form>

Common Form Elements

  • <input type="text"> - Single line text box for short inputs
  • <input type="email"> - Email address input (browsers validate the format)
  • <input type="password"> - Password field (hides what you type with dots or asterisks)
  • <textarea> - Multi-line text area for longer messages
  • <button> - Submit button that users click to send the form
  • <label> - Labels that describe each input field
Note: Forms need backend programming (like PHP or JavaScript) to actually send the data somewhere. For now, focus on creating the form structure in HTML. We'll learn about handling form data later.

9. Semantic HTML Elements

Semantic HTML means using tags that clearly describe their meaning. Instead of using generic <div> tags for everything, we use descriptive tags that explain what the content is. This makes your code more meaningful and helpful.

Why Use Semantic HTML?

Semantic tags help with:

  • Search engines understand your content better and rank you higher
  • Screen readers navigate your site more easily for people with visual impairments
  • Other developers understand your code quickly and easily
  • Maintainability - Your code becomes easier to update and modify

Common Semantic Tags 

<header>
    <h1>My Website</h1>
    <nav>
        <a href="home.html">Home</a>
        <a href="about.html">About</a>
        <a href="services.html">Services</a>
    </nav>
</header>

<main>
    <article>
        <h2>Article Title</h2>
        <p>Article content goes here...</p>
    </article>
</main>

<footer>
    <p>Copyright 2025 My Website</p>
</footer>
  • <header> - Top section of the page, usually containing logo and navigation
  • <nav> - Navigation menu with links to different pages
  • <main> - Main content area of the page
  • <article> - Self-contained content like blog posts or news stories
  • <section> - Thematic grouping of content
  • <aside> - Content related but separate from main content (like a sidebar)
  • <footer> - Bottom section with copyright, contact info, or links

10. Practice Your Skills

Now it's your turn! Try creating a simple webpage that includes everything you've learned in this course. This hands-on practice is the best way to solidify your HTML knowledge.

Challenge: Build Your First Website

Create a webpage that includes all of these elements:

  • Correct HTML markup containing head and body parts
  • One main title (h1) and a number of titles (h2)
  • Not less than three paragraphs of the text
  • Text styling (bold, italic, or underline) in one paragraph at least
  • Link to an external website at least once
  • Link to another page on the same site at least once
  • One or more pictures with suitable alt text
  • Ordered or unordered list (or both) present
  • A simple table of 3 rows and 3 columns minimums
  • A contact form with 3 input fields minimum
  • Use of Semantic HTML (header, main, footer) elements

Tips for Success

  • Begin with a basic setup and slowly progress. Avoid jumping into the advanced stuff right away.
  • Regularly check your code in a browser to monitor the visual output.
  • Perfection is not the thing here and rather the practice and learning is the goal.
  • Indent your code properly to make it neat and easy to read.
  • Insert comments in your code to describe the function of each part.
  • Error-making is very natural and recognized as a learning method!
Next Steps: When you feel at ease with HTML and have practice project already done, it is time for you to learn CSS! CSS is the tool that gives you artistic control over your websites and allows you to make them look the way you want. You will be taught about colors, fonts, layouts, and a lot more!
